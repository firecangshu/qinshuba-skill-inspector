# 秦叔宝 v2.0 升级提案（PR Draft）

> 提案人：赛博女娲 v1.0.1 作者（基于 SkillHub 实战反馈）
> 提案时间：2026-07-05
> 当前版本：v1.0.5.1
> 目标版本：v2.0.0

---

## 概述

本次提案基于**赛博女娲 v1.0.1 在 SkillHub 平台发布的实战反馈**。秦叔宝当前评分 4.92/5.00 (A 级)，但 SkillHub 平台仍然拒绝了 3 次提交，原因如下：

1. **不允许的文件类型**（`.gitignore` / `LICENSE` / `CHANGELOG.md`）
2. **版本号格式不符**（`1.0.1` 不被 SkillHub 接受，要求 `1.0.0`）
3. **平台必填字段缺失**（`category` / `platforms`）

这暴露了秦叔宝的**核心盲点**：只审计**内容安全**，不审计**平台兼容**。

---

## 1. 背景

### 1.1 当前秦叔宝 v1.0.5.1 检查维度

| 维度 | 范围 |
|------|------|
| **T** Trust | 可信任度（7 项）|
| **R** Reliability | 可靠性（7 项）|
| **A** Adaptability | 适用性（7 项）|
| **C** Convention | 规范性（9 项）|
| **E** Effectiveness | 有效性（7 项）|
| **S** Security | 8 维安全审计 |

### 1.2 实战证据（赛博女娲 v1.0.1）

| 检查项 | v1.0.5.1 评分 | SkillHub 实际反馈 |
|--------|--------------|------------------|
| 内容安全 | ✅ A 级 4.92 | ✅ 通过 |
| 平台文件白名单 | ❌ 未检查 | ❌ 拒绝 3 次 |
| 平台元数据必填 | ❌ 未检查 | ❌ 缺 category / platforms |
| 平台版本号格式 | ❌ 只查 semver | ❌ 不接受 1.0.1 |
| zip 结构 | ❌ 未检查 | ⚠️ 可能被拒 |
| 大小限制 | ❌ 未检查 | ⚠️ 可能被拒 |

**结论**：秦叔宝评分与平台实际接受度脱节。

---

## 2. 改进方案

新增 **P（Platform Compatibility）维度**，作为第 6 个评分维度。

### 2.1 P 维度 6 项检查

#### P01. 目标平台文件白名单

**问题**：不同平台允许的文件类型不同。

**改进**：

| 平台 | 允许 | 拒绝 |
|------|------|------|
| SkillHub | SKILL.md / README.md / references/ / scripts/ | .gitignore / LICENSE / CHANGELOG.md / .git/ / __pycache__/ |
| GitHub | 全部 | 无 |
| Claude Code | SKILL.md | 其它可放但用不到 |
| Cursor | SKILL.md | 其它可放但用不到 |

#### P02. 目标平台元数据必填字段

**问题**：不同平台必填字段不同。

**改进**：

| 平台 | 必填 | 选填 |
|------|------|------|
| SkillHub | name, version, category, platforms, description | author, license, trigger |
| GitHub | (无必填) | 全部可选 |
| Claude Code | name, description | version |
| Cursor | name, description | version |

#### P03. 目标平台版本号格式

**问题**：不同平台版本号规则不同。

**改进**：

| 平台 | 严格度 | 示例 |
|------|--------|------|
| SkillHub | 严格 X.Y.Z | `1.0.0` ✅ / `1.0` ❌ / `1.0.0.0` ❌ / `1.0.1` ❌（实际）|
| GitHub | 自由 | 任意 |
| 通用推荐 | semver | `1.0.0` |

**注**：SkillHub 实战显示仅 `1.0.0` 占位符被接受，可能需要进一步验证。

#### P04. zip 根目录结构

**问题**：解压后目录结构是否正确。

**改进**：
- ✅ 推荐：解压后根目录 = skill 名（如 `cyber-nuwa/`）
- ❌ 拒绝：嵌套目录 / 文件散落根目录

#### P05. 大小限制

**问题**：不同平台对 skill 大小有限制。

**改进**：

| 平台 | 限制 |
|------|------|
| SkillHub | < 1 MB（推荐）|
| GitHub | < 100 MB |
| Claude Code | < 10 MB（推荐）|

#### P06. 发布前预检清单

**问题**：用户发布前不知道该准备什么。

**改进**：按目标平台生成检查清单：

```markdown
## 发布到 SkillHub 前的预检清单

- [ ] 文件全部在 SkillHub 白名单内
- [ ] 元数据必填字段齐全（name / version / category / platforms / description）
- [ ] 版本号格式为 X.Y.Z（推荐 1.0.0）
- [ ] zip 解压后根目录 = skill 名
- [ ] 大小 < 1 MB
- [ ] description ≤ 50 字
- [ ] 触发词明确，不与其他 skill 冲突
- [ ] 通过三线安全审核
```

## 3. 实施细节

### 3.1 在 Phase 0 加"目标平台"问询

```markdown
### Phase 0.5: 目标平台确认

**问题**：你要把这个 Skill 发布到哪个平台？

| 选项 | 平台 | 规范要求 |
|------|------|---------|
| A | SkillHub（国内）| name + version + category + platforms，文件白名单严格 |
| B | GitHub | name 可选，文件自由 |
| C | Claude Code / Cursor | 仅 name + description |
| D | 其他 | 用户指定 |

**对应检查项**：
- 选 A → 启用 P01-P06 全套平台检查
- 选 B → 仅启用 P01（GitHub 允许所有文件）
- 选 C → 启用 P01（仅 SKILL.md）
- 选 D → 用户提供平台要求
```

### 3.3 新增 scripts/platform_check.py

```python
#!/usr/bin/env python3
"""
platform_check.py - 平台兼容性检查脚本
按目标平台检查 skill 是否符合发布要求
"""

import yaml
import zipfile
import os

PLATFORM_RULES = {
    "skillhub": {
        "required_fields": ["name", "version", "category", "platforms", "description"],
        "allowed_files": ["SKILL.md", "README.md", "references/", "scripts/"],
        "forbidden_files": [".gitignore", "LICENSE", "LICENSE.md", "CHANGELOG.md",
                            ".git/", "__pycache__/", ".DS_Store"],
        "version_pattern": r"^\d+\.\d+\.\d+$",
        "description_max_length": 50,
    },
    "github": {
        "required_fields": [],
        "allowed_files": ["*"],
        "forbidden_files": [],
    },
    "claude_code": {
        "required_fields": ["name", "description"],
        "allowed_files": ["SKILL.md"],
    },
}

def check_platform(skill_path, platform):
    """检查 skill 是否符合指定平台规范"""
    rules = PLATFORM_RULES.get(platform, {})
    results = []

    # P01 文件白名单
    if "forbidden_files" in rules:
        forbidden = find_forbidden_files(skill_path, rules["forbidden_files"])
        if forbidden:
            results.append({
                "check": "P01",
                "level": "ERROR",
                "message": f"不允许的文件: {', '.join(forbidden)}"
            })

    # P02 元数据必填
    if "required_fields" in rules:
        frontmatter = parse_frontmatter(skill_path)
        missing = [f for f in rules["required_fields"] if f not in frontmatter]
        if missing:
            results.append({
                "check": "P02",
                "level": "ERROR",
                "message": f"缺失必填字段: {', '.join(missing)}"
            })

    # P03 版本号格式
    if "version_pattern" in rules:
        import re
        version = frontmatter.get("version", "")
        if not re.match(rules["version_pattern"], version):
            results.append({
                "check": "P03",
                "level": "ERROR",
                "message": f"版本号格式不符: {version}（要求 {rules['version_pattern']}）"
            })

    return results
```

### 3.4 评分模型变化

**v1.0.5.1**：
```
综合分 = T×0.25 + R×0.20 + A×0.20 + C×0.15 + E×0.20
```

**v2.0**：
```
综合分 = (T×0.20 + R×0.20 + A×0.15 + C×0.10 + E×0.20) × 平台系数

平台系数：
- P01-P06 全过 = 1.0（满分）
- 1 项 FAIL = 0.85
- 2 项 FAIL = 0.70
- ≥3 项 FAIL = 0.50
- 跳过平台检查（未指定目标平台）= 0.90
```

## 4. 测试验证

### 4.1 案例 1：赛博女娲 v1.0.1

**v1.0.5.1 评分**：
- 综合分：4.92（A 级）
- 警告：无
- 实际：SkillHub 拒绝 3 次

**v2.0 评分**（假设指定 SkillHub 目标）：
- T/R/A/C/E：4.92（不变）
- P01 FAIL：.gitignore
- P02 FAIL：缺 category / platforms
- P03 FAIL：1.0.1 不符合 X.Y.Z 严格格式
- 平台系数：0.50（3 项 FAIL）
- 综合分：4.92 × 0.50 = **2.46（D 级）**
- 警告：**无法在 SkillHub 发布，需修复 P01-P03**

**v2.0 评分**（修复后）：
- P01-P03 全过
- 平台系数：1.0
- 综合分：4.92 × 1.0 = **4.92（A 级）**
- 警告：无
- 实际：✅ 成功发布

### 4.2 对比表

| Skill | v1.0.5.1 评分 | v2.0 评分（修复前）| v2.0 评分（修复后）| 实际发布 |
|-------|--------------|------------------|------------------|---------|
| 赛博女娲 v1.0.1 | 4.92 A | 2.46 D + 警告 | 4.92 A | ✅ |
